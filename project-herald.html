Herald Examiner MR

Computer Vision as a Narrative System

Overview

What happens when computer vision starts narrating the physical world? Herald Examiner MR is a mixed reality prototype where detected objects become anchors for storytelling. Built for Meta Quest 3 using Unity and Unity Sentis, the experience transforms the user's environment into a dynamic, time-shifting narrative space.

Concept

Set inside the historic Herald Examiner Building — now home to ASU's Emerging Media campus — the experience explores how AI perception can reinterpret reality. Real-world objects are detected and transformed into narrative artifacts: a laptop becomes a 1926 newsroom typewriter, a book becomes a March 1926 Herald Examiner newspaper, and the physical space becomes a living archive across time. The user moves through three eras: 1926, the 1960s, and the present day. Each transition reshapes the environment through visual tone, sound, and embodiment.

Interaction Design

The system uses computer vision as the core interaction layer. Objects are detected in real time, bounding boxes become spatial anchors, and virtual content attaches directly to physical context. Embodiment cues reinforce presence: historical wristwatches per era, spatial narration with subtitles, and passthrough filters shifting time periods. The result is a system where the world itself becomes the interface.

Key Innovation

Computer vision as a storytelling trigger, not just utility Physical environment reinterpreted as a narrative canvas Seamless blending of AI detection + spatial anchoring Time travel conveyed through perception shifts, not UI

Technical Highlights

Unity + Meta Quest 3 mixed reality passthrough. Real-time object detection with Unity Sentis. Spatial anchoring from 2D bounding boxes → 3D world space. Dynamic passthrough styling across 1920s, 1960s, and modern eras. Spatial audio narration with subtitles. Performance-aware anchoring system.

AI-Assisted Workflow

Design concepts translated into dev prompts via ChatGPT and Claude. Prompts executed in-engine via Cursor. Scripts and tools generated directly inside Unity. Narration generated with ElevenLabs. Unity console feedback loop used to refine systems. Goal: move toward a fully automated iteration loop between AI and engine.

Key Technical Insight

Attaching CGI to moving detected objects caused major instability — bounding boxes update every frame, introducing jitter and performance spikes. Conclusion: static object anchoring is significantly more reliable for narrative experiences on-device.

Research Direction

If AI can understand our environment, can it turn reality into a story? Next step: AI-directed volumetric actors via Gaussian Splatting, moving through real space and interacting dynamically with detected objects.
