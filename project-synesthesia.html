Spatial Synesthesia

Perception as Interface in Spatial Computing

Overview

What if looking at color changed the way you hear music? Inspired by Wassily Kandinsky and his encounter with Arnold Schoenberg, Spatial Synesthesia explores a new interaction paradigm for spatial computing — where visual perception becomes the interface. Built for Apple Vision Pro using visionOS, the experience transforms gaze into a conductor of sound.

Interaction Concept

Instead of buttons or gestures, the system responds directly to what you look at: Yellow → Brass section emerges. Blue → Strings take over. No focus → Full orchestra returns. At the same time, the physical environment shifts color to match the soundscape, blending perception across real and virtual space.

Design Challenge

Both the painting and the symphony are abstract. This led to a core UX question: How do you teach users a system without telling them how it works?

Experience Design

The interaction unfolds in two phases: First, the system teaches you — color and sound are introduced together, gradually building a mental model. Then perception takes over — the user naturally discovers control. Gaze becomes the only input. No UI. No gestures. No instructions.

Key Innovation

Eye-tracking as a primary interaction layer Environment-wide color immersion via passthrough Transition from learning to intuitive control Interaction based on perception, not intention

Technology

visionOS + RealityKit + SwiftUI. Real-time gaze raycasting and color sampling. Spatial audio mixing with dynamic instrument isolation.

Workflow

A rapid AI-assisted prototyping pipeline: UX concept → AI translation → Code execution. Design ideation, prompt-to-code translation via Claude, implementation via Cursor in Xcode. This workflow enabled fast iteration on interaction models and behavior tuning.

Reflection

As spatial computing becomes embedded in physical space: Should interfaces be explicitly taught… or naturally discovered through perception?
